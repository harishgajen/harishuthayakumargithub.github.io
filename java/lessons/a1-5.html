Arrays of Compound Data Types

Concept | Examples | Java Implementation

Getting Started: This lesson will cover the use of arrays of compound data types. So far, you have created arrays using primitive data types and object data types, but arrays of compound data types can also be made (data types with multiple values within one).

Recall: Arrays with Primitive Data Types

Up to this point, the arrays we have used have been integer arrays and String arrays among a few others. While int is a primtiive data type and String is an object data type, both data types hold a single value. However, compound data types are data types with multiple values/variabels within that single initialized data structure.

To recap, here is how to initialize arrays of non-compound data types: 

                                        // The number in the square brackets at the end of the initialization lets Java know that your array is 5 elements long.
String[] names = new String[5]; 
int[] numbers = new int[5];

Two-Dimensional Arrays

An array is a compound data type itself, since each array holds multiple values within it. For that reason, an array of arrays, also called a 2D array, can be made.

Here is the format of initializing a 2D array: int[][] abc = new int[5][4];. The only difference in the syntax of initializing a regular integer array is the presence of an additional pair of square brackets. The purpose of the additional round brackets is to provide the size of that "second dimension" brackets is to provide the size of that second dimension. For example, to create a 2D integer array made up of 5 arrays, with 6 elements in each of those arrays, it would be initialized with the following statement: int[][] arr = new int[5][6];

Like with normal arrays, the index starts at 0. Therefore, the format to print the 4th element in the 2nd array would be: System.out.println(arr[1][3]); Here, 1 is the index of the second array and 3 is the index of the 4th element in that second array.

Another important concept is traversing a 2D array. With one dimensional arrays, a singular for loop is sufficient to traverse the array, and print out all the elements, for example. This task is done using a nested for loop. The outer for loop is used to traverse the arrays within the array, and the inner for loop is used to traverse the elements within each array that the outer for loop is currently on. Another way of conceptualizing this, is that the outer for loop starts from the first row and the inner for loop traverses each element in that row by going through each column. This process is then repeated with each row/column combination.

Example:
Note 1: Instead of initializing the 2d array with a set number of arrays and set number of elements within each array, this example initializes the array with values within each. This allows for each array in the 2d array to have different sizes.
Note 2: The condition for the inner for loop is arr[r].length to check the length of the specific array within the 2d array that it is currently checking. Since in this example, each array has a variable length, this ensures that all elements are printed out.
Note 3: Inside the nested for loop, System.out.print() is used instead of System.out.println() so that the elements within the same array are printed on the same line. 

int [][] arr = {{12, 5, 6, 1},
			    {28, 311, -5, 6},
				{-2, 15, 27}};
		// r for row, c for column
		for (int r = 0; r < arr.length; r++) { 
			for (int c = 0; c < arr[r].length; c++) {
				System.out.print(arr[r][c] + " ");
			}
			System.out.println(); // moving to next line
		}

'Less/Greater Than' Comparisons (< and >)

Firstly, the char data type can use < and > just like other primitive data types. For example: 'A' < 'b' is a valid comparison. Every character has a corresponding integer value, which is what is actually compared in the background when comparing chars. 'B' is considered to be less than 'a' since capital letters come before lower case numbers - 'B' has a smaller corresponding integer valuie than 'a' does.

To compare Strings, the comparable interface has another method called compareTo(). This method is called using the same format as the equals() method. This method returns an integer, which can be negative, 0, or positive. To explain what the returned value means, lets say that String name1 = "Bob" and String name2 = "Janet" If we were to order these two names in alphabetical order, we know that "Bob" comes first. Likewise, if we called the compareTo method like so: name1.compareTo(name2); the number returned would be a negative. Therefore, we can conclude that when the String used to call the method (name1 in this example) is less than the String it is compared to (name2), a negative number is returned. On the other hand, if the comparison made was name2.compareTo(name1); a positive number would be returned, since name2 comes after name1, making name2 larger. Finally, if the two Strings being compared are equal, 0 is returned. This could technically be used to check if two Strings are equal, but the equals() method in the subsection above is easier to use.
