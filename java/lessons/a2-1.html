Intro to Modular Programming

Concept | Examples | Java Implementation

Getting Started: This lesson will cover the basics of modular programming, a large topic that we will be covering in the next few lessons.

What is Modular Programming?

Modular programming is a software development technique where the functionality of a program is separated into independent modules. What this means is that when you are creating a program, you import different modules and libraries that each have an independent feature that you want to integrate into your program.

This is good practice, because different components of your program that function independently are separated into independent files. If you were to change the code of one Java class for example, it would not impact the other components that are in other files. This lesson will be covering integrated libraries and modules and how they interact with user-defined classes. Believe it or not, you likely have already done modular programming to some extent.

Integrated Libraries & Modules and Import Statements

When creating a Java program that requires user input, you are likely used to importing Scanner by adding the following line to the top of your Java file: import java.util.Scanner; Now, whatever code is written in this file cannot impact the original code of the Scanner class. Now, some functions in Java are readily available in every class you make, such as System.out.println(), so you may be wondering why others have to be imported as well as why we can't import all modules at once.

Well, the reason why not all modules are readily available without imports are to avoid a cluttered namespace. For example, if you imported all modules and then wanted to create a function called 'router', it is possible that one of the modules you imported already uses that name. Now, you are forced to lookup all the taken names and you are more limited. To avoid all this confusion, it is much easier to only import the modules that are needed.

Introductino to Objects and Instances

Another great benefit to modular programming is the ability to create multiple instances of one particular object. For example, Strings in Java are instances of the String class. Everytime you initialize a String, it creates an instance of the String class. It creates a String object that has all the built-in functions and features of the String class. When you create another String variable, it creates another instance of a String, which is completetely independent from the previous one. Being able to create multiple instances of a single class is very convenient. Imagine only being able to have one string in each Java program - you would constantly have to reuse the same variable and wouldn't be able to have two strings at once.

Look forward to the next lesson, we will start to create our own classes to use in object oriented programming. Everything with the Scanner class and String class mentioned in this lesson, can also be done with your own object classes!
